SETUP

縦動画プロジェクトを作る

最初の1回だけ丁寧に。2本目以降はテンプレート複製で30秒で終わります。

新規プロジェクト → 解像度を縦に変更

  1. 起動後のプロジェクトマネージャーで New Project → 名前を付けて作成
  2. 右下の歯車アイコン(Project Settings)を開く
  3. Master Settings → Timeline resolution を Custom にして 1080 × 1920 と入力
  4. Timeline frame rate は 30(または素材に合わせて 60)。ここだけは後から変更できないので素材撮影時のfpsを先に確認
  5. Save で確定
つまずきポイント フレームレートは「クリップを1本でもタイムラインに置いた後」はグレーアウトして変更不可になります。iPhone撮影素材は30fpsか60fpsが多いので、読み込む前に素材を1つ右クリック→情報で確認しておくのが確実です。

横素材しかない場合 — Smart Reframe STUDIO限定

横16:9の素材を縦9:16タイムラインに置くと左右が切れますが、Studioなら被写体を自動追尾してフレーミングしてくれます。

  1. タイムライン上のクリップを選択
  2. 右上 Inspector → Video → Smart Reframe を開く
  3. Object of Interest を Auto にして Reframe を押す
MEDIA

素材を読み込む

凝った管理は不要。ドラッグ&ドロップだけ覚えれば十分です。

  1. 画面下部のページタブから Edit(または Cut)ページを開く
  2. FinderからMedia Pool(左上の領域)へ動画・音声・画像をドラッグ&ドロップ
  3. 「フレームレートが違う」と警告が出たら Don't Change を選ぶ(プロジェクト設定を維持)
つまずきポイント Resolveは素材を「リンク」で参照します。読み込み後に元ファイルを移動・削除するとクリップが赤くなり再リンクが必要に。素材は1つのフォルダにまとめてから読み込む運用にすると事故ゼロです。例: ~/Movies/shorts/2026-06-11_案件名/
EDIT

カット編集 — 作業の8割はここ

ショート動画の編集は実質「不要部分を削る」だけ。以下の5操作で完結します。

覚えるショートカットは5つだけ

操作キー用途
再生 / 停止Space確認
早送り・巻き戻しJ K LL連打で倍速。チェックが速い
カット(分割)⌘B再生ヘッド位置でクリップを切る
削除して詰める選択して右クリック →
Ripple Delete		隙間を残さず削除
取り消し⌘Z何回でも戻れる

基本の流れ

  1. Media Poolのクリップをタイムラインへドラッグ
  2. L 連打の倍速再生で全体を流し見
  3. 不要部分の頭で ⌘B、終わりで ⌘B
  4. 挟まれた部分を選択 → 右クリック → Ripple Delete
  5. クリップの端をドラッグすれば微調整(トリム)も可能
CutページとEditページ、どっちを使う?

結論: Editページだけで始めてOK。

Cutページは高速編集特化(2画面タイムライン等)で慣れると速いですが、操作体系が独特で解説情報もEditページ前提が大半。学習コストを考えると、まずEditページに一本化し、月10本以上量産する段階になったらCutページを試すのが効率的です。

テンポを出すコツ(ショート動画特有)
  • 冒頭2秒に結論かフックを置く。前置きカットは全削除
  • 文と文の間の「えー」「間」を ⌘B×2 + Ripple Delete で刻む(ジャンプカット)。違和感はテロップが吸収してくれる
  • 尺はYouTube Shorts/TikTok/Reelsとも60秒以内に収めれば3つ同時投稿できる
TEXT

テロップ・自動字幕

ショート動画は音なし視聴が多く、字幕の有無で視聴維持率が変わります。Studioなら自動生成できます。

音声から字幕を自動生成 STUDIO限定

  1. メニューバー Timeline → Create Subtitles from Audio
  2. 言語で Japanese を選択 → Create
  3. タイムライン最上部に字幕トラックが生成される
  4. 字幕クリップをダブルクリックすれば誤認識をその場で修正可能
  5. Inspector → 字幕トラックの Track Style でフォント・サイズ・縁取り・位置を一括変更
つまずきポイント 字幕トラックはそのままだと書き出しに焼き込まれない設定の場合があります。Deliverページで Export Subtitle → Burn into video にチェックが入っているか必ず確認(後述の書き出しプリセットに含めれば以後は自動)。

タイトル・強調テロップを手動で入れる

  1. Editページ左上 Effects → Titles → Text+ をビデオトラック2(クリップの上段)へドラッグ
  2. Inspectorでテキスト入力。サイズ・色・縁取り(Shading)を設定
  3. セーフエリア: 上部はアプリのUI、下部はキャプション欄と被るため、テロップは中央〜やや下に配置
毎回同じデザインにする(テロップの使い回し)

調整済みのText+クリップを右クリック → Create New Fusion Generator…でも保存できますが、より簡単なのはスタイル済みText+をタイムラインに置いたままプロジェクトごとテンプレ化する方法(自動化の章を参照)。コピー&ペーストで文字だけ打ち替えれば、デザイン作業が毎回ゼロになります。

AUDIO

音声とBGM

Fairlightページは開かなくてOK。EditページのInspectorだけで足ります。

声を聞きやすくする最短手順

  1. 声のクリップを選択 → Inspector → Audio
  2. Voice Isolation をON STUDIO限定 — 環境音・ノイズをAIが除去。これだけで品質が一段上がる
  3. 音量はクリップ中央の白い横線を上下ドラッグ。声は再生時にメーターが-12〜-6dBあたりで振れる程度が目安

BGMを敷く

  1. BGMファイルをオーディオトラック2へドラッグ
  2. 音量線を下げて-25〜-20dB程度に(声の邪魔をしない)
  3. 終端のフェードアウト: クリップ右上角の白い▲マーカーを左へドラッグ
つまずきポイント BGMの著作権はプラットフォーム側で検出されます。TikTok/Reels内の公式音源はアプリ内でしか使えないため、Resolveで編集する場合はYouTube Audio Libraryや商用可のフリー音源を使うこと。
DELIVER

書き出し — 3プラットフォーム共通設定

YouTube Shorts / TikTok / Reels は同一ファイルで投稿可能。プリセットを1回作れば以後ワンタップです。

設定値(1回だけ手入力)

  1. 画面下部タブから Deliver ページへ
  2. 左上 Render Settings で Custom Export を選択
  3. 以下を設定:
項目値
FormatMP4
CodecH.264(互換性最優先)
Resolution1080 × 1920
Frame rateタイムラインと同じ(30 or 60)
QualityRestrict to 20,000〜25,000 Kb/s
Audio CodecAAC / 320 Kb/s
Subtitle(自動字幕使用時)Export Subtitle → Burn into video
  1. 保存先を指定 → Add to Render Queue
  2. 右側のキューで Render All → 完了

プリセット化(2回目以降を10秒に)

上記設定の状態で Render Settings 右上の 「…」メニュー → Save As New Preset。「ショート縦1080p」等の名前で保存すれば、次回からプリセット選択 → Add to Queue → Render の3タップで終わります。

AUTO

自動化の足がかり

「全自動編集」は現実的でないが、毎回の繰り返し作業をゼロにする仕込みは今日からできます。効果順に3段階。

段階1: プロジェクトテンプレート(即効・効果大)

縦設定 + スタイル済みテロップ + BGMトラック + 音量設定まで作り込んだプロジェクトを1つ用意し、プロジェクトマネージャーで右クリック → Duplicate(複製)から毎回スタート。セットアップ時間が毎回15分 → 0分に。

段階2: プリセットの積み上げ

  • レンダープリセット(前章) — 書き出し設定の再入力をゼロに
  • Power Bin: Media Poolで右クリック → Show Power Bins。ここに入れた素材(ロゴ・定番BGM・効果音)は全プロジェクト共通で使い回せる
  • 字幕Track Styleのプリセット保存 — テロップデザインの統一を自動化

段階3: スクリプティングAPI(Hermes連携の入口)

ResolveはPython/Luaの公式スクリプティングAPIを持ち、外部からプロジェクト作成・素材読み込み・レンダー実行を叩けます。確認方法:

  1. メニュー Workspace → Console でPy3を選ぶとAPIを対話実行できる
  2. APIドキュメントはMac内の /Library/Application Support/Blackmagic Design/DaVinci Resolve/Developer/Scripting/ にREADMEとサンプルが同梱

現実的な最初の自動化対象は「素材フォルダを監視 → 新規プロジェクト生成 → 素材読み込み → タイムライン配置まで自動」。編集判断は人間、準備と書き出しは機械、という分担が費用対効果の最適点です。Mac mini側のHermes Agentからスクリプトを起動する構成も組めます(外部スクリプティングはStudio版のみの機能なので、その点でもStudioを持っている強みが活きます)。
